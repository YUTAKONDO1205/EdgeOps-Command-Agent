作るべきもの

EdgeOps Command Agent

センサー・画像・点検記録・マニュアルを統合し、異常検知から保全判断・作業指示・報告書作成まで自動で回す現場保全AIエージェント



一言でいうと、



「異常を見つけるAI」ではなく、異常発見後に“現場が次に何をすべきか”まで決めるAgent



です。



近藤さんの研究である Spresense × 振動・音響センサ × 異常検知AI を核にしつつ、ハッカソンの要件である Microsoft Azure / Microsoft AI / Agentic AI / 業務改革 に寄せます。



1\. そもそものハッカソン要件



まず、このハッカソンで最低限満たさないといけない条件は明確です。



公式ルール上、開発物は Azure App Service、Azure Container Apps、Azure Functions、AKS、Azure VM、Azure GPU/AI VM などの Microsoft Azure アプリケーション実行基盤、または Copilot Studio のマネージド実行環境を使う必要があります。さらに、Microsoft AI 技術として Microsoft Foundry / Azure OpenAI、Copilot Studio、Azure AI Agent Service、Semantic Kernel、Azure AI Vision / Speech / Language / Translator、Fabric系AI機能 などを1つ以上使う必要があります。



提出物は、審査員が実際に試せる 成果物URL、Zennの アプリ紹介記事、記事内に埋め込む 3分程度のデモ動画、アーキテクチャ図 が必須です。GitHubリポジトリは任意ですが、出せるなら公開しておいた方が技術力の証明になります。



評価軸は、公式には次の3つです。



ビジネスインパクト

業務課題を的確に捉え、その解決がどれほど大きな価値を生むか。

アプローチの有効性

Agentic AIとしての振る舞いやアーキテクチャが、課題解決に対して論理的か。

完成度・実現性

成果物が安定して動作し、実際の業務導入を想定した実装コストや運用性が考慮されているか。



つまり、ただの「ChatGPTっぽい便利アプリ」では勝てません。

勝つには、業務課題が深い、Agentがちゃんと動く、Azureで安定稼働する、実務導入が想像できる ことが必要です。

これはかなり強いです。
